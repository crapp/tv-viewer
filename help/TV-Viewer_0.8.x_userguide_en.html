1. System Requirements

Minimum requirements:
  • The card has to be configured correctly and you will require preinstalled ivtv-tune as well as v4l2-ctl. Both of them are included with the latest ivtv drivers, on most systems these tools are part of the package ivtv-utils.
  •  TCL/TK Version 8.5 upwards has to be preinstalled. In most cases it is supplied by the distros buildservice. Alternatively cou could use ActiveTcl.
  • The integrated TV playback is based on MPlayer, therefor you need to have installed MPlayer >= 1.0rc2. A most recent version is recommended.
Additional requirements:
  • For being able to view the help dialogues within the program, you will need the "xdg-utils". These should be default content of most recent todays distros, too. More information about xdg-utils can be found here.
  • If you have installed Tcl/Tk < 8.6 you need the package "tkimg" so the GUI can use png images for the icons. If tkimg is not installed TV-Viewer will use gif images instead, these do not have the same quality than png images. On Debian based systems this package is called libtk-img.
  • If you want to make use of LIRC functions in combination with your remote control and the GUI, you can realize this via the named package.
    For more informations about LIRC and the GUI, refer to the user guide section 4. Remote controls.

2. Installation

If there are no packages for your distribution, download the packed files and extract them into a folder of your choice. After that you should find a file named "tv-viewer_install.sh".
Run this file in a terminal as superuser (root).

> sudo ./tv-viewer_install.sh

or:

> su
# ./tv-viewer_install.sh

This command will install TV-Viewer in "/usr/local/share/tv-viewer", the configuration files are located in "~/.tv-viewer".
Use "sudo tv-viewer_install.sh --uninstall" to deinstall TV-Viewer.
Type in "tv-viewer" (not as root!) to start the programm. If you encounter any problems run the diagnostic routine with "tv-viewer_diag" and file a bug report on sourceforge.net.

The installation script creates an entry in your main menu under "Multimedia/TV".

After the installation you should check the preferences and create a station list.

3. General Usage


3.1 Main frontend

Menu bar: The menu bar provides access to all functions of TV-Viewer. E.g: Preferences, Station Editor, Color Management, User Guide ...

Timeshift: This feature is disabled in version 0.8.1a1. It will be available in version 0.8.1

Record: Starts the record wizard.

EPG: Starts the chosen program to retrieve the television program.

Start TV: Start/Stop TV playback, this button is disabled if the application can not find a working installation of MPlayer or if there is an active recording.


3.2 Preferences


3.2.1 General

Here you may set general options for TV-Viewer.

Language: Choose your language, if you are not happy with the Language that was automatically chosen.

Start TV on startup: If checked launches TV playback on start up.

Newsreader: The Newsreader will check for news on the TV-Viewer project. You can define the interval in days.

Electronic Program Guide: Choose a program which you want to use to retrieve the television program. E.g. "firefox http://www.tvguide.com/Listings/"


3.2.2 Analog

Video Device Node: Choose the appropriate video device node. Use dmesg together with grep if you're not sure which node you have to use.

Video Standard: Set the video standard and frequency table. Depends on where you are located. Force Standard can be used if the driver is choosing the wrong video standard.

Video Input: Defines the default video input TV-Viewer should make use of. With the Station Editor you can define the video input for each station.
Note you can also change the video input with "Ctrl-I" / "Ctrl-Alt-I".

Analog Stream Settings: Here you may define some preferences for the mpeg stream provided by your TV-Card. Don't set the videobitrate (peak) to a lower value than videobitrate and vice versa. Temporal filter may be usefull if the image appears blurred. Hardware volume control defines the volume level of the videostream. For all these options the standard values are recommended.


3.2.3 DVB

TV-Viewer will support DVB devices with version 0.9.x


3.2.4 Video

The options here refer to MPlayer. The tooltips provide basic informations on all available options. You will find a description here.


3.2.5 Audio

The audio output driver refers to MPlayer. Alsa is recommended. Use the software mixer so the volume change doesn't affect system wide volume levels. The option "Audio channels" is used by MPlayer so the audio is decoded to the chosen value.


3.2.6 Radio

Radio support will be integrated in version 0.8.2


3.2.7 Interface

Theme: Choose your preferred theme for TV-Viewer.

Tooltips: Enable/Disable tooltips for the different parts of TV-Viewer.

Splash Screen: If checked the splash screen will be displayed on initialization.

Station List: Use this if you want the staion list to be shown on initialization.

Window Properties:

Video window:
The first option makes sure, that there will be no separate entry in taskbar for the video window. On the other hand it will be logically linked to the main window. For example the video window will also dock to the system tray regardless of the option "Dock video window".

Start TV window in full-screen mode.

Station list alignment: Choose whether the station list should be aligned left or right when video window is in full-screen mode.

System Tray:
Dock video window: If you dock TV-Viewer to the system tray this option determines whether the video window should be docked as well or not.

Dock TV-Viewer after initialization: Docks TV-Viewer after the initialization to the system tray.

Miinimize to Tray: Whenever the main window is minimized it will be docked to the system tray.

On screen Display: Displays volume level, zoom factor, current video input and station name in the video window.


3.2.8 Record

Standard folder: Define the standard folder for your recordings.

Default record duration: Specify the default record duration. This value is taken when you add a new recording. Format: hh:mm:ss Example: 02:30:00 (Duration is 2 hours 30 minutes and 0 seconds)

Allow station change while recording: This option will allow you to change stations although there is a running recording.

Autostart scheduler: Activate this option if the scheduler should be started as a daemon when you boot up your computer. This option is important if you schedule recordings and should be activated.


3.2.9 Advanced

Additional options for MPlayer: Provide additional command line options for MPlayer, as well as extra video and audio filters. See the MPlayer manual for more infos about audio and video filters and other available options.

Logs: Enable or disable logging of MPlayer, TV-Viewer and Scheduler events. This can be very usefull to narrow down problems. If you want to file a bug report on sourceforge.net this logs should be attached. Error and warning messages will be displayed bold in the logviewer.


3.3 Station Editor

The Station Editor can be used to create a station list.

Station search: Executes the station search gui. There you may choose on which video input you want to search for stations, append the new stations to the existing list or perform a full frequency sweep.

Add: Append a new station. You will have to provide a name, frequency and video input.

Delete: Remove the selected station

Edit: Edit an existing station.

(De)Activate: Enable or disable stations. Disabled stations will be marked red.

Up / Down: Change the order of the stations.

Preview: Start tv playback to preview the selected station. This may be usefull to query the station name or to adjust the frequency.

Note, you may choose more than one station by using control or shift keys.


3.4 Color Management

Adjust brightness, hue, saturation according to your needs. These values are global values.

3.5 Record Wizard

The Record Wizard is a gui to schedule and manage recordings.

New recording: Here you may add a new recording. First you should choose the station to record.
  • Time: Specify when the recording should start (24hours clock). If you omit this value the recording will start immediately.
  • Date: Select the date on which the recording should occur. If not specified the recording will occur on the current date. Format: yyyy-mm-dd
  • Duration: Set the duration of the recording. Format: hh:mm:ss Example: 02:30:00
  • Resolution: Video resolution. If you don't specify a resolution the standard resolution will be used. In most cases it is not necessary to change this value. It is better to use some software like avidemux afterwards to shrink the video.
  • Ouput file: Define the output path and file. If not set, the file will be stored in the default folder for recordings.
Delete: Delete one or more recording(s).

Edit: Edit an existing recording.

Every time you add, edit or delete a recording the scheduler will be stopped and restarted automatically. If you schedule a recording and restart your computer the Scheduler won't start automatically unless you set the appropriate option in the preferences.
The Scheduler stops automatically if there are no more pending jobs.
You can see the status of the Scheduler and active recordings in the status frame. Use the button "Stop recording" to stop recordings prematurely.

After a recording has started and TV-Viewer has been started only file-playback mode will be available. In this mode you may watch the recording. As soon as the recoring has stopped you may start TV playback which will quit file-playback mode.

4. Remote controls

TV-Viewer can be controlled by your remote control. In order to use this feature you'll have to install the Lirc remote control drivers.

First of all make sure you execute "irexec -d" when you boot up your computer.

You will need the file ".lircrc" in your users home directory.

Possible actions for your remote control:
  • teleview (Start/Stop TV playback)
  • station_up (Zap one station up)
  • station_down (Zap one station down)
  • station_jump (Jump between the last two stations)
  • key_[0-9] (Keys 0 - 9 to choose station number)
  • fullscreen (Toggle fullscreen)
  • quit (Close TV-Viewer)
  • zoom_incr (Increase zoom)
  • zoom_decr (Decrease zoom)
  • zoom_wide (Autozoom 16:9)
  • zoom_stnd (Standard zoom)
  • record (Start Record Wizard)
  • volume_incr (Increase volume)
  • volume_decr (Decrease volume)
  • mute (Toggle mute)
The following actions can only be used if you are watching a recording.
  • pause (Toggle pause)
  • stop (Stop file playback - Meanwhile recording will continue)
  • start (Start file playback)
  • forward_10s (10 seconds fast forward)
  • forward_1m (1 minute fast forward)
  • forward_10m (10 minutes fast forward)
  • forward_end (End of the file)
  • forward_start (Beginning of the file)
  • rewind_10s (10 seconds rewind)
  • rewind_1m (1 minute rewind)
  • rewind_10m (10 minutes rewind)
  • rewind_start (Beginning of the file)
Now an example configuration for your ".lircrc":

##################################
begin
      remote = Hauppauge #Name of the remote control
      prog = irexec
      button = TV #The name of the button which you want to use to start tv-viewer.
      config = tv-viewer & #Execute TV-Viewer, don't forget the "&"
      mode = tv-viewer #If you use an extra mode for TV-Viewer your remote control won't affect any other programs.
      repeat = 0 #Enable or disable repeat
    end

begin tv-viewer #Marks the begin for mode tv-viewer.

begin
      remote = Hauppauge
      prog = irexec
      button = Channel_UP
      config = tv-viewer_lirc station_up #Zap one station up.
      repeat = 0
    end

begin
      remote = Hauppauge
      prog = irexec
      button = Channel_DOWN
      config =tv-viewer_lirc station_down #Zap one station down.
      repeat = 0
    end

begin
      remote = Hauppauge
      prog = irexec
      button = Back/Exit
      config = tv-viewer_lirc quit #Quit TV-Viewer
      flags = mode #Exit mode tv-viewer
      repeat = 0
    end

end tv-viewer #Marks the end  for mode tv-viewer.
########################################

For more options see the list above.
